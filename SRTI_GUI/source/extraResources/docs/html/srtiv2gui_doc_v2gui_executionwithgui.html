SRTI v2 GUI - Execution (with GUI):

This section covers details how to run a simulation system from within the SRTI v2 GUI.


How To Run:

  1. First, click on "File -> Export Execute Files" in the Menu Bar.
  2. Click the POWER button in the Execute Bar, and wait a few seconds. You should see an instance of the RTI Server (with the Java-based Server GUI) open, and it should indicate that all expected simulators have connected (this includes simulators that occur in any stage, not necesasrily just stage 0). After a few seconds, the PLAY and STOP buttons should become enabled. If launching a simulator in a large program (example: Matlab), wait until everything has launched, not just for when the PLAY button becomes available.
      3. If the RTI Server or simulators have not launched or connected as expected, some further debugging may be required outside the SRTI v2 GUI.
  3. Click the PLAY button. It should tell the RTI Server to start the system, and the simulators will begin to run. Inside the SRTI v2 GUI, you should see simulators on the Simulator Canvas begin to change color, cycling from red to blue. The Inspector Panel should indicate the current timestep of the simulation system, and a Feed of the most recent messages for you to analyze (each can be expanded to show their contents).
      4. If a simulator changes red and stays red, that means the system has halted, waiting for that simulator to "finish" the current step.
  4. At any time, you can click the PAUSE button to temporarily pause the active system. You can resume by clicking PLAY again.
  5. When you choose, you can click the STOP button to end execution and close the RTI Server and all active simulators. If an "End Condition" is satisfied, it will cause this event automatically.
      6. When closing these processes, it is normal for error and warning prompts to occur. Generally, these can be ignored.

When To Use This Feature:

Being able to launch and control your SRTI simulation system from one location is one of the major benefits of the SRTI v2 GUI. For most simulation systems using SRTI v2.00.00, we recommend giving this a try.

However, if any errors occur, it can be difficult to debug the root cause from the SRTI v2 GUI. If this becomes a problem, we recommended running your system without the SRTI v2 GUI first, and to use the GUI for subsequent runs when it is confirmed to work. See HTML Documentation for "SRTI v2.00.00 - Execution (without GUI)" for more information.
