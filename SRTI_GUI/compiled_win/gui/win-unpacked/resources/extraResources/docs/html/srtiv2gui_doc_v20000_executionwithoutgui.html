SRTI v2.00.00 - Execution (without GUI):

This is a reminder of the steps to take to start an SRTI v2.00.00 system. Installing the Java Runtime Environment (JRE) is the minimum requirement to run the SRTI and its components. Further details about language-dependent commands can be reviewed in the HTML documentation under "SRTI v2.00.00 - RTI Wrapper" and "SRTI v2.00.00 - User Simulations and Language Differences."


Ordered Steps to Execute:

You can generally follow these steps to manually launch an SRTI v2.00.00 system.

  1. Start the RTI Server.
    • Download and double-click on SRTI_v2_22_02.jar. A small window should appear listing the hostname and portnumber.
    • To set certain configuration details for the RTI Server, download "settings.txt" and place in the same file directory.
  2. Launch each individual simulator.
    • If you haven't already done so, edit and prepare the appropriate configuration files for the RTI Wrapper. This includes ensuring the port number to connect to the RTI Server is correct.
    • Download and place the appropriate-language RTI Wrapper in the same directory as your simulator file(s).
    • Run the RTI Wrapper. Depending on the Wrapper you use, you may need to launch it through a command prompt / terminal.
    • You should see each simulator listed in the Server GUI when they each connect. If not, check for any errors from either the RTI Wrapper or RTI Server. Generally, it is recommended to restart the RTI Server if something fails.
  3. When all simulators have been connected, click on the "Send Start Request To All" button on the Server GUI.
    • You should see the Server GUI updating with new messages.
  4. If you wish to pause / resume the simulation system temporarily, click on the "Send Pause Request To Server" button.
  5. To end the simulation system, close the Server GUI.
    • Each connected simulator should quit automatically, or throw an error and remain open, depending on the language. Check and close each simulator manually if necessary to avoid ghost processes.

Running each simulator manually can be tedious for large simulation systems (as an example, if you have 10 simulators, you could need to have 10 command line / terminal windows active). The SRTI v2 GUI helps manage and automate this process, but relies on these core instructions, which can be replicated on a variety of systems, including remote or cluster servers.
