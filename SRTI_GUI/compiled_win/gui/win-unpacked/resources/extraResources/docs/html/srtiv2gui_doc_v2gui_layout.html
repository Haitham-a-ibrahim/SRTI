SRTI v2 GUI - Layout:

This section explains the basic layout of the SRTI v2 GUI.


Layout Map:

  1. Menu Bar: Access to functions like "New Project," "Save Project," "Export Execute FIles", "Undo" and "Redo."
  2. Action Mode Bar: Changes the action the mouse cursor does [select object, configure/edit object, create publish/subscribe connection, delete object].
  3. Execute Bar: For a finished project, allows the user to Start, Play, Pause and Stop an SRTI v2.00.00 simulation system (launches the set of system commands required to open the RTI Server and individual simulators).
  4. Object List Panel: List of simulator and message definitions saved for this project. Can create a new definition, or import an existing definition from a previous project.
  5. Object (Simulator) List Panel: A list of simulators that can be added to the canvas. User must define a simulator here first to be used in the project.
  6. Object (Message) List Panel: A list of messages that can be added to the canvas. User must define a message here first to be used in the project.
  7. Simulator Canvas: A canvas that visually defines the order of execution of simulators in a given stage. Also defines publish/subscribe connections.
  8. Message Canvas: A canvas that visually defines message channels that a simulator in the project can publish or subscribe to.
  9. Inspector Panel: When selecting an object in the v2 GUI in "configure/edit object" mode, optional windows to edit certain properties show up in this panel.
  10. Stage Bar: The ability to add multiple stages within the same simulation system execution, which might consist of different simulators, different publish/subscribe connections and other properties.
  11. Action Output Box: A debug text-box to confirm events as they are accomplished, including the currently-open project, and confirming undo/redo actions.
