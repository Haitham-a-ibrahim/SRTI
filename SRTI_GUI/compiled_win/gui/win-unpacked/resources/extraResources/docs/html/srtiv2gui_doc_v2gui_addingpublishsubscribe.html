SRTI v2 GUI - Adding Publish / Subscribe:

This section outlines the steps to add a publish / subscribe connection between a simulator and a message in the SRTI v2 GUI.


Steps:

  1. After having placed a simulator and message on the canvas, change to "connection" mode.
  2. To add a "publish" connection: First, click on a simulator. Then click on the message you want it to publish to. A window will appear asking you to set configuration variables that define this connection.
      3. To undo a selection, you can click anywhere on the canvas that isn't a simulator or message.
  3. The "Publish Details" window is similar to the "Subscribe Details" window. You will see a list of variables that exist in the message, and will have the option to choose a variable of the same datatype from the simulator to correspond to that variable (this means the value in the simulator will be copied to the message at the time the message is published). There are additional variables that determine when and how often this message should be published by this simulator; see the HTML Documentation for "SRTI v2.00.00 - RTI Configuration File" for a better understanding of these properties.
  4. After clicking "Confirm," the window will close, and you will see a red box with an arrow leading from the simulator to the message.
      5. If you want a second chance to edit this connection, you can click on the red box in "Configure/Edit" mode.
  5. To add a "subscribe" connection: First, click on a message. Then click on the simulator you want to subscribe to that message. A window will appear asking you to set configuration variables that define this connection.
      6. To undo a selection, you can click anywhere on the canvas that isn't a simulator or message.
  6. The "Subscribe Details" window is similar to the "Subscribe Details" window. You will see a list of variables that exist in the simulator, and will have the option to choose a variable of the same datatype from the message to correspond to that variable (this means the value in the message will be copied to the simulator at the time the message is accessed). There are additional variables that determine when and how often this message should be published by this simulator; see the HTML Documentation for "SRTI v2.00.00 - RTI Configuration File" for a better understanding of these properties.
  7. After clicking "Confirm," the window will close, and you will see a red box with an arrow leading from the message to the simulator.
      8. If you want a second chance to edit this connection, you can click on the red box in "Configure/Edit" mode.
  8. Remember: A simulator can publish and subscribe to multiple messages. A message can be published by multiple simulators, and subscribed to by multiple simulators.
