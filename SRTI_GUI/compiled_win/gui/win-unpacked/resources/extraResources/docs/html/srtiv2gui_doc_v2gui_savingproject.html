SRTI v2 GUI - Saving Project:

This section covers details how the SRTI v2 GUI saves its project files.


How To Save:

To save a project, go to "File -> Save" or "File -> Save As" in the Menu Bar, and follow prompts to name your project if necessary, and to choose a folder location. If successful, a notication will appear indicating the file has been saved. After saving, you can reopen the project through "File -> Open."


What Do The Save Files Look Like?:

There are 3 types of files that get output when saving a project:

  • 1 ".project" file
  • 'n' ".simdef" files
  • 'n' ".mesdef" files

All of these files are saved in ASCII-string JSON format, and can be viewed in any text editor.

The .project file is the main file, and it represents all information about your project. The additional files are not required to reopen or transfer a project.

The .simdef files represent each simulator definition you added to your project. The data in this file is a portion of the information already stored in the .project file. The reason these are generated is to allow you to easily transfer simulator definitions for use in other projects or for other users (you can add a new simulator to a project using the "Import" option and selecting a .simdef file).

The .mesdef files serve the same purpose as the .simdef files, but represent each definition of a mesasge in the project rather than simulators.


Exporting Files For Execution:

Under "File" in the Menu Bar, there is also an option called "Export Execute Files."

When you make and save a project using the SRTI v2 GUI, you've only made a visual representation. When you click "Export Execute Files," you generate and output the actual files required for a simulator system to run. This includes the configuration files an RTI Wrapper relies on, and the "settings.txt" file for the RTI Server. Each file is generated in the same folder location as the RTI Wrapper and RTI Server in the project.

Before trying to run a project, remember to both save your project and to export excute files for it to behave as expected.
